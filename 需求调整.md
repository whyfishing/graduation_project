# 需求调整说明（聚焦网易云单平台）

## 1. 调整背景

由于当前可稳定获取的数据源受限，原方案中的“多平台统一分析”在本阶段不具备完整落地条件。  
因此本轮调整为：**先完成网易云音乐单平台的数据采集、分析与可视化闭环**，其他平台保留为后续扩展目标。

## 2. 调整后的范围

### 2.1 本阶段范围（In Scope）

1. 数据平台固定为网易云音乐。  
2. 页面主导航聚焦三个业务域：排行榜、歌单、歌手。  
3. 每个业务域支持“推荐列表 -> 详情歌曲列表 -> 分析展示”。  
4. 对歌曲补充标签（tags），用于分类、画像与相似推荐。  
5. 支持听众样本采集与听众画像分析。  
6. 支持歌单画像、歌手画像、排行榜画像展示。

### 2.2 暂缓范围（Out of Scope）

1. 多平台切换的实际数据联动能力。  
2. 其他平台（QQ音乐、酷狗、抖音等）的采集与对齐。  
3. 企业级高并发、分布式容灾能力。  

## 3. 前端交互调整

### 3.1 顶栏

- 排行榜  
- 歌单  
- 歌手

### 3.2 左侧边栏

左侧边栏基于顶栏一级菜单展示二级功能，例如：

1. 排行榜：榜单分类、榜单详情、榜单画像。  
2. 歌单：推荐歌单、歌单详情、歌单画像。  
3. 歌手：歌手分类、歌手详情、歌手画像。  

### 3.3 右侧分析栏

右侧增加画像与分析区，展示：

1. tags 分布（饼图/柱状图）。  
2. 文本画像摘要。  
3. 相似歌曲推荐。  

## 4. 数据来源页面（网易云）

1. 排行榜主页：https://music.163.com/#/discover/toplist  
2. 歌单主页：https://music.163.com/#/discover/playlist  
3. 歌单样例：https://music.163.com/#/playlist?id=17745989905  
4. 歌手主页：https://music.163.com/#/discover/artist  
5. 歌手样例：https://music.163.com/#/artist?id=10559  

## 5. 关键业务逻辑

1. 点击排行榜/歌单/歌手推荐项，可进入对应歌曲列表。  
2. 歌曲以网易云 song_id 作为唯一标识。  
3. 歌手以网易云 artist_id 作为唯一标识。  
4. 听众以网易云 user_id 作为唯一标识。  
5. 歌曲标签通过歌词调用 AI 生成并持久化，用于分类与画像。  
6. 听众样本可从评论区用户切入，再进入用户主页提取最近听歌记录。  

## 6. 数据库设计调整结论

不建议“排行榜/歌单/歌手各自一张冗余歌曲大表”，建议采用“**实体主表 + 关系映射表 + 画像结果表**”：

1. 主实体表：songs、artists、playlists、users。  
2. 关系映射表：rank_songs、playlist_songs、artist_songs、user_recent_songs。  
3. 分析结果表：song_tags、user_profiles、playlist_profiles、artist_profiles、rank_profiles。  

这样可避免同一歌曲重复存储，支持后续多维分析和功能扩展。

## 7. 最小字段建议

### 7.1 songs

| 字段 | 说明 |
|---|---|
| song_id | 网易云歌曲ID（主键） |
| song_name | 歌曲名 |
| duration_sec | 时长（秒） |
| album_name | 专辑名 |
| lyric_text | 歌词文本 |
| song_url | 歌曲URL |
| source_platform | 固定为 netease |

### 7.2 artists

| 字段 | 说明 |
|---|---|
| artist_id | 网易云歌手ID（主键） |
| artist_name | 歌手名 |
| artist_url | 歌手URL |

### 7.3 playlists

| 字段 | 说明 |
|---|---|
| playlist_id | 网易云歌单ID（主键） |
| playlist_name | 歌单名 |
| playlist_url | 歌单URL |

### 7.4 核心映射关系

1. playlist_songs(playlist_id, song_id, position)。  
2. rank_songs(rank_id, song_id, rank_position, crawl_time)。  
3. artist_songs(artist_id, song_id)。  

### 7.5 标签与画像

1. song_tags(song_id, tag_name, score, source_model, updated_at)。  
2. user_profiles(user_id, profile_json, updated_at)。  
3. playlist_profiles(playlist_id, profile_json, updated_at)。  
4. artist_profiles(artist_id, profile_json, updated_at)。  
5. rank_profiles(rank_id, profile_json, updated_at)。  

## 8. 实施优先级

### 8.1 Must

1. 网易云单平台采集（排行榜/歌单/歌手基础信息）。  
2. 歌曲与关系映射入库。  
3. 页面完成“推荐列表 + 歌曲列表 + 基础分析图表”。  

### 8.2 Should

1. 歌曲标签自动生成。  
2. 听众样本采集与基础画像。  
3. 歌单、歌手、排行榜画像。  

### 8.3 Could

1. 相似歌曲推荐增强。  
2. 多平台统一入口UI占位。  

## 9. 风险与说明

1. 网易云页面结构变化会影响采集稳定性，需要字段容错。  
2. 评论区与用户主页数据可见性可能受限制，需要样本策略。  
3. AI 标签成本与稳定性需控制，建议缓存与批处理。  
4. 本文档作为当前阶段调整基线，后续扩展需增补版本记录。
